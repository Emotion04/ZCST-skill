# 贡献指南

## 角色说明

本项目有两个角色，使用两套逻辑：

| 角色 | 职责 | 工具 |
|------|------|------|
| **贡献者** | 提交 PR 改进知识库 | 任何能获取信息的工具 |
| **维护者** | 审核 PR、定期更新文档、采集闭源平台信息 | kimi-webbridge |

普通用户用 `/zcst` 提问时只走本地文档，不需要装任何额外工具。

## 贡献方式

### 提 Issue

适合不想直接改文档的情况：发现信息过时、错误、缺失，或有新信息来源推荐。

Issue 标题请注明对应分类，如 `[餐饮] 旧饭二楼新开了奶茶店`。

### 提 PR

1. Fork 本仓库
2. 修改 `docs/` 下的对应文件
3. 确保每条信息有来源标注
4. 提 PR 到 `main` 分支

## 信息验证标准

不同信息类型有不同的可信度要求：

| 可信度 | 信息类型 | 验证要求 |
|--------|---------|---------|
| **高** | 校历、政策、官方流程 | 必须有学校官网/公众号原文链接 |
| **中高** | 设施信息（开放时间、位置） | 至少 2 人确认，或附照片佐证 |
| **参照** | 生活经验（美食推荐、穿衣建议） | 标注来源平台（小红书/贴吧/本人），注明为经验分享 |
| **主观** | 「最好吃」「最方便」等评价 | 只标注为「个人推荐」，多人推荐可注明共识程度 |

### 来源标注格式

文档中每条信息应标注来源和日期：

```markdown
| 档口 | 推荐 | 价格 | 来源 |
|------|------|------|------|
| 螺蛳粉 | 原味螺蛳粉 | 12元 | 小红书 @小红薯 2026-06 |
```

对于官方信息，保留原文链接：

```markdown
新生报到时间由学校官方发布（[来源](https://zsb.zcst.edu.cn/)），请以最新公告为准。
```

## PR Review 清单

Reviewer 会用以下标准审核：

- [ ] 信息来源是否标注清楚
- [ ] 日期是否标注
- [ ] 官方信息是否有公告链接
- [ ] 主观评价是否注明为个人观点
- [ ] 修改范围合理（不夹带无关改动）

## 开发者

### 安装 kimi-webbridge

kimi-webbridge 用于采集小红书、微信公众号等 WebSearch 搜不到的内容。

```bash
参考 [kimi-webbridge 官方网站](https://www.kimi.com/features/webbridge)安装

安装后验证
~/.kimi-webbridge/bin/kimi-webbridge status
```

### 数据采集流程

1. 确定采集目标（如「小红书珠科餐饮最新推荐」）
2. 用独立 session name 导航到目标页面
3. snapshot 读取正文
4. 整理到对应 `docs/*.md`，标注来源和日期
5. 关闭 session
6. 提 PR

### 每学期维护清单

| 时间 | 文档 | 检查项 |
|------|------|--------|
| 8 月初 | `academics.md` | 新学年校历、新生报到日期 |
| 8 月初 | `enrollment.md` | 新生入学须知是否有更新 |
| 8 月 | `dormitory.md` | 宿舍分配政策变动 |
| 8 月 | `dining.md` | 新增/关闭的餐饮店铺 |
| 12 月 | `academics.md` | 转专业申请时间、寒假日期 |
| 6 月初 | `graduation.md` | 离校手续流程更新 |
| 全年 | 所有文档 | 关注 Issue 中报告的错误和过时信息 |
